# 申請帳號

CNTUG Infra Labs 開放所有希望使用的人申請。本文說明申請流程、所需資訊、審核與帳號生命週期，幫助您順利取得 OpenStack 環境的存取權限。

## 申請表單

請填寫官方申請表單：[Infra Labs 申請表單](https://forms.gle/Hx8oxCBWAFZWUD1Z7)

提交後，管理員會在審核通過後，由自動化系統建立 OpenStack 使用者與專案，並透過 Email 寄發含有初始密碼的歡迎信。

## 需準備的資訊

填寫表單前，請先想清楚以下欄位內容：

| 欄位 | 說明 | 範例 |
|---|---|---|
| 姓名 | 您的真實姓名，便於管理員辨識 | 王大明 |
| 使用者名稱 | OpenStack 登入用的帳號名稱，建議使用英文小寫，將同時作為您的專案名稱 | `daming` |
| Email | 接收歡迎信、到期通知的信箱 | `daming@example.com` |
| 使用用途 | 說明您打算如何使用 Infra Labs 資源 | 開源專案測試、研究實驗、技術社群分享 |
| 使用時間 | 帳號有效期限，目前提供 `兩週`、`一個月`、`三個月`、`六個月` 四種選項 | `三個月` |
| 使用條件 | 簡述您與開源社群的關聯（例如貢獻程式碼、撰寫文件、社群活動等） | 貢獻程式碼至開源專案 |
| vCPU 數量 | 預估需要的 vCPU 數，留空則套用預設配額 | `2` |
| 記憶體 (GB) | 預估需要的記憶體容量，留空則套用預設配額 | `4` |
| 儲存空間 (GB) | 預估需要的磁碟容量，留空則套用預設配額 | `40` |
| 其餘設備 | 是否需要 Load Balancer、GPU 或物件儲存等額外資源 | `Load Balancer, GPU` |

:::tip 配額設定
若您不確定資源需求，建議將 vCPU、記憶體、儲存空間欄位留空，系統會套用預設配額。日後若需調高配額，可參考 [FAQ](/docs/faq) 聯絡管理員另行申請。
:::

## 審核流程

提交申請後，您的資料會經歷以下流程：

```
申請送出 ──> 管理員審核 ──> 自動化系統建立資源 ──> 收到歡迎信
```

1. **管理員審核**：管理員會檢視申請理由與資源需求，確認無誤後將表單狀態標記為 `approved`。
2. **資源建立**：自動化系統每日定期執行，偵測到 `approved` 狀態後會在 OpenStack 上建立對應的使用者、專案、配額與角色（包含 `member`，若申請 Load Balancer 會額外加上 `load-balancer_member`）。
3. **歡迎通知**：建立完成後，系統會寄發歡迎信給您，內容包含初始密碼以及登入方式。`infra@cloudnative.tw` 會被加入 CC，未來若您回覆此信件可直接聯繫管理員。

審核通常會在數天內完成。若您遲遲未收到通知，可以寄信至 [infra@cloudnative.tw](mailto:infra@cloudnative.tw) 詢問進度。

## 收到帳號後

歡迎信中會附上：

- OpenStack Horizon 登入網址：[https://openstack.cloudnative.tw](https://openstack.cloudnative.tw)
- 您的使用者名稱與初始密碼
- 文件、Telegram 社群、狀態頁等相關連結

收到帳號後，請立即依下列順序進行初始設定：

1. 依照 [開始使用 OpenStack](/docs/intro) 的步驟登入並更換初始密碼。
2. 上傳 SSH 金鑰、建立安全群組與 VM，請依序參考[基礎教學](/docs/category/基礎教學)章節。

## 帳號生命週期與到期通知

每個帳號都有期限（依您在表單上選擇的使用時間決定）。系統會根據以下時程通知您：

| 階段 | 動作 |
|---|---|
| 到期前 14 天 | 寄送到期預警信，提醒您資料備份或申請延展 |
| 到期當日 | 帳號標記為已過期，但資源尚未刪除 |
| 過期後 | 由管理員確認後，才會進入刪除流程 |

:::warning 資料備份
帳號到期後，OpenStack 上的所有資源（VM、Volume、網路、物件儲存 Bucket 等）都會被刪除。請務必在到期前自行備份重要資料。
:::

### 申請延展

若您仍需要繼續使用，請於到期前主動聯絡 [infra@cloudnative.tw](mailto:infra@cloudnative.tw)，並提供延展原因與預計使用期限。
